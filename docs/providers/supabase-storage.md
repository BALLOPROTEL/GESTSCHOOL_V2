# Supabase Storage Provider

Supabase Storage is the durable document storage target for production/staging. Local storage remains a dev/test descriptor fallback only.

Official references:
- https://supabase.com/docs/reference/javascript/storage-from-createsigneduploadurl
- https://supabase.com/docs/reference/javascript/storage-from-createsignedurl
- https://supabase.com/docs/guides/storage/serving/downloads

## Runtime Ownership

Render API only:
- `STORAGE_PROVIDER=supabase`
- `FILE_STORAGE_DRIVER=SUPABASE`
- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `SUPABASE_STORAGE_BUCKET_DOCUMENTS=gestschool-documents`
- `SUPABASE_STORAGE_BUCKET_RECEIPTS=gestschool-receipts`
- `SUPABASE_STORAGE_BUCKET_REPORT_CARDS=gestschool-report-cards`
- `SUPABASE_STORAGE_BUCKET_AVATARS=gestschool-avatars`
- `SUPABASE_STORAGE_SIGNED_UPLOAD_TTL_SECONDS=7200`
- `SUPABASE_STORAGE_TIMEOUT_MS=10000`

Never configure `SUPABASE_SERVICE_ROLE_KEY` in Vercel.

## Buckets

Required buckets:
- `gestschool-documents`
- `gestschool-receipts`
- `gestschool-report-cards`
- `gestschool-avatars`

Recommended bucket policy:
- private buckets
- uploads only through signed upload URLs generated by the backend
- downloads only through backend-generated signed URLs or authenticated backend mediation

## Path Convention

Documents:
- `tenants/{tenantId}/students/{studentId}/documents/{fileName}`

Receipts:
- `tenants/{tenantId}/receipts/{invoiceId}/{fileName}`

Report cards:
- `tenants/{tenantId}/report-cards/{studentId}/{schoolYearId}/{fileName}`

Avatars:
- `tenants/{tenantId}/avatars/{userId}/{fileName}`

## Endpoint

`POST /api/v1/storage/upload-descriptor`

The endpoint returns:
- storage driver
- bucket
- object key
- signed upload URL
- expiry

It must never return `SUPABASE_SERVICE_ROLE_KEY`.

## Config Check

Use the secure monitoring endpoint after deploy:

```bash
curl -H "x-metrics-token: $MONITORING_METRICS_TOKEN" \
  https://gestschool-ylik.onrender.com/api/v1/monitoring/providers
```

Expected storage section:
- `provider=SUPABASE`
- `enabled=true`
- each required Supabase variable is `true`

The endpoint does not validate bucket existence. A real upload descriptor smoke test is still required.

## Real Smoke Test

1. Call `POST /api/v1/storage/upload-descriptor`.
2. Confirm `driver=SUPABASE`, bucket and key are present.
3. Upload a tiny validation file with the signed upload URL.
4. Confirm the object exists in Supabase.
5. Delete the validation object from Supabase dashboard or a controlled admin script.

If this is executed from local without Render env variables, do not mark Supabase as validated.

## Production Guard

`FILE_STORAGE_DRIVER=LOCAL` is blocked when `NODE_ENV=production` unless `FILE_STORAGE_ALLOW_LOCAL_IN_PROD=true` is explicitly set. That override is for emergency only and must not be used as final storage strategy.

## Rollback

1. Stop uploads if Supabase is failing.
2. Switch `FILE_STORAGE_DRIVER=LOCAL` only outside production, or use an emergency pre-signed webhook provider.
3. Keep object keys already issued; do not delete buckets during rollback.
4. Reissue upload descriptors after the provider is restored.
